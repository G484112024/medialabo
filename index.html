「Webコンテンツ作成」の成果

本テーマについて

このテーマでは，HTML, CSS, JavaScript の基本を学び，これらを組み合わせて Web ページを構成する方法を学びます．

学習目標：

  • HTML によって Web ページの構造を記述する基本的な方法がわかる
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる
  • HTML, CSS, JavaScript の構文リファレンスから，構文の書式や使用方法の情報を入手できる．
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる

なお，Webページのデザインに関する事柄や画像・音声・動画の編集に関連する事柄はこのテーマの範囲外とします．

プロジェクト課題について

このサイトは、世界の天気情報を簡単に確認できるシンプルな検索サイトです。ユーザーは画面上部のプルダウンメニューから都市を選択し、「表示」ボタンをクリックすることで、その都市の現在の天気をすぐに確認することができます。現在は北京市に対応しており、選択すると気温や天候などの情報が画面に表示されます。使いやすさを重視したシンプルなインターフェースで、誰でも直感的に操作できるのが特徴です。天気APIと連携することで、常に最新の天気データが取得されるようになっており、海外の天気を調べたいときに便利なツールとして活用できます。

プロジェクト課題で作成したページ：project.html

数当てゲーム

このWebページは、1から10までの整数を当てる「数当てゲーム」ができるシンプルなブラウザゲームです。ユーザーは入力欄に数字を入力し、「回答」ボタンをクリックすることで予想を送信します。画面には現在の試行回数が表示され、 予想した数が正しいかどうかの判定結果がリアルタイムで表示されます。JavaScript（kazuate.js）と連携しており、正解か不正解かを自動的に判定し、その結果をページ上に反映する仕組みになっています。正解するまで何回でも挑戦でき、ゲーム感覚で楽しく数字の予想を楽しむことができます。簡単な操作とわかりやすい表示により、子どもから大人まで幅広いユーザーが気軽に楽しめる内容となっており、プログラミング学習やインタラクティブなUI設計の入門としても活用できるサイトです。

数当てゲームのページ：kazuate.html

各授業回の学習事項

第1回 WWW と HTML

HTMLを学ぶことで、ウェブページの構造を理解し、自分でサイトを作成できるようになります。タグや要素の使い方を覚えると、情報を整理しやすくなり、他のウェブ関連技術の学習にも役立ちます。

演習の成果:演習1-4: さまざまな要素

第2回 CSSによるWEBページの装飾

CSSを学ぶことで、ウェブページの見た目を自由にデザインできるようになります。色やレイアウト、フォントなどを調整することで、より魅力的で使いやすいサイトが作れます。HTMLと組み合わせることで、表現力が大きく広がります。

演習の成果:演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

JavaScriptの基本は、Firefoxのウェブ開発ツールの「コンソール」機能を使って学ぶことができます。コンソールは、コードを直接入力して即座に実行結果を確認できる便利なツールです。たとえば、console.log("こんにちは");と入力すれば、"こんにちは"と表示されます。変数の宣言や関数の定義、条件分岐や繰り返し処理なども試すことができ、プログラムの動作をリアルタイムで確認できます。これにより、JavaScriptの理解が深まります。

演習の成果:演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

この授業では、HTMLでフォーム要素を設置し、JavaScriptによるイベント処理とDOM操作を学ぶことで、ボタンを押すとページが変化する動的なWebページの作成方法を習得します。

演習の成果:演習4-10:足し算

第5回 イベント処理

今回は、DOM を用いて要素の作成、属性の追加・変更、削除を行います。JavaScript を使い、Web ページの内容を動的に操作する方法を学びます。

演習の成果:演習5-7 DOMプログラミング

本プロジェクトでは、天気情報をユーザーが選択した都市に応じて取得・表示するシンプルかつ実用的なWebアプリケーションを目指しました。工夫点としてまず挙げられるのは、都市選択用のプルダウンメニューの導入です。これにより、ユーザーは直感的に都市を選べるようになっています。 また、project.cssやstyle.cssでのビジュアル調整にも工夫を凝らしました。特に背景にはlinear-gradientを使い、視覚的に動きが出るようにしています。project.cssでは青系のグラデーションで天気の涼しげなイメージを表現し、style.cssでは白とオレンジの配色で明るく温かみのある印象に仕上げています。 HTML構造はシンプルに保ちつつ、CSSのclassやIDをうまく使い分けて、モジュール的にデザインを適用できるように設計しました。例えば、div#div3～div#div5には共通のスタイルを適用しつつも、それぞれ独立してカスタマイズが可能なように配慮しました。

特に苦労したのは、ページのデザインとCSSレイアウトの調整です。見やすく整った画面を目指し、背景にはlinear-gradientを使ってグラデーションを加えましたが、文字とのコントラストが弱くなり、何度も色や角度を試して調整しました。 また、div#div3〜div#div5の各ブロックに個別のIDを設定し、統一感を保ちつつ柔軟に編集できるようにしましたが、スタイルの重複や変更時の手間が増え、メンテナンス性に注意が必要でした。 画像の位置調整でも苦戦しました。float: rightを使った際に他の要素と重なることが多く、paddingやmarginを細かく調整する必要がありました。 さらに、複数のCSSファイルを併用していたため、スタイルの競合や優先順位の問題も発生しました。これを通じて、CSSの特異性やカスケードルールについて理解が深まりました。

最も強調したいのは、ユーザー体験を意識したインターフェース設計です。例えば、「都市を選択してください」という初期のプルダウン表示や、ボタンを押したときに明確に反応があるように設計した点は、ユーザーにとって操作が分かりやすいよう意図しました。 また、デザインの一貫性も重視しました。CSS内で使っている色（#3f3f3fなど）やボーダーのスタイルを統一することで、見た目に統一感を持たせ、情報の視認性を高めることを心がけました。

本プロジェクトを通じて、HTML・CSS・JavaScriptそれぞれの役割や連携について理解を深めることができました。特に外部APIとの連携という点では、実用的なWeb開発において必要不可欠な技術であり、学習しがいがありました。 また、CSSによるデザインの細部調整には多くの時間を費やしましたが、そのぶん完成したときの達成感も大きかったです。今後はさらにJavaScriptの機能を拡張し、複数都市の同時比較表示や、天気アイコンの自動表示などの機能も追加してみたいと感じました。 今後の改善点としては、レスポンシブ対応やアクセシビリティの向上、さらにはコードの可読性を高めるためのリファクタリングなどが挙げられます。しかしながら、今回のプロジェクトはシンプルでありながらも実用的で、基本をしっかりと学べるよい経験となりました。
